
Paperless-ngx Troubleshooting and Setup Documentation

1. Verify Paperless Services:
   To ensure all services are running correctly, execute the following:
   ```
   systemctl list-units | grep paperless
   ```
   Expected output should show:
   - paperless-consumer.service
   - paperless-scheduler.service
   - paperless-task-queue.service
   - paperless-webserver.service

2. Check Logs for Errors:
   To debug issues, inspect the logs of relevant services:
   - For scheduler:
     ```
     journalctl -u paperless-scheduler.service
     ```
   - For consumer:
     ```
     journalctl -u paperless-consumer.service
     ```
   - For webserver:
     ```
     journalctl -u paperless-webserver.service
     ```
   - For task queue:
     ```
     journalctl -u paperless-task-queue.service
     ```

3. Verify Configuration Directory and Files:
   Use the `find` command to locate configuration or critical files:
   ```
   find /opt/paperless -type f -name "*.env"
   ```
   Example output should confirm the location of environment files.

   Additionally, verify the existence of critical directories:
   ```
   ls -ld /opt/paperless/data
   ```

4. Recreate celerybeat-schedule.db (if needed):
   If the `celerybeat-schedule.db` is causing issues:
   ```
   find /opt/paperless -type f -name "celerybeat-schedule.db"
   rm -f /opt/paperless/data/celerybeat-schedule.db
   touch /opt/paperless/data/celerybeat-schedule.db
   systemctl restart paperless-scheduler.service
   ```
   After restarting, confirm it is running:
   ```
   systemctl status paperless-scheduler.service
   ```

6. Manually Train the Classifier:
   To retrain the classifier:
   ```
   python3 /opt/paperless/src/manage.py document_create_classifier --verbosity 3
   ```
   Verify that the updated model is saved:
   ```
   find /opt/paperless/data -type f -name "classification_model.pickle"
   ```

   Expected output should confirm the file's existence.

7. Verify Web Access:
   Ensure the Paperless web interface is accessible at its configured IP/hostname. Use a browser to check:
   ```
   http://<your-server-ip>:8000
   ```

8. Restarting Services:
   To ensure changes are applied or services are reset after an error:
   ```
   systemctl restart paperless-consumer.service
   systemctl restart paperless-scheduler.service
   systemctl restart paperless-task-queue.service
   systemctl restart paperless-webserver.service
   ```

This document provides a comprehensive guide to troubleshoot, verify, and configure your Paperless-ngx instance. Always validate each step and adapt commands to your specific setup.
